## Observations

- Claude handled a very long prompt without losing context.
- Sequential input collection improved the quality of the generated report.
- Structured headings and tables made the report easier to understand.
- Career and wealth analysis was the most detailed section.
- Breaking the report into logical sections improved readability.

---

## Key Learnings

- Well-structured prompts produce significantly better outputs.
- Collecting complete user information before reasoning reduces ambiguity.
- Clearly defined output sections help Claude maintain consistency.
- Tables improve readability for long analytical reports.
- Prompt engineering is as important as model capability for complex tasks.
